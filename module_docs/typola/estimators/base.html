typola.estimators.base

Estimator base class + evaluation utilities.

An estimator is just a callable counts -> probabilities with a name and a params dict. Making it a small dataclass-like object (rather than a bare function) lets us carry configuration, log runs reproducibly, and build an evaluation harness that compares strategies.

class typola.estimators.base.Estimator(name: str = 'estimator', params: ~typing.Mapping = <factory>)[source]

Callable that maps count vectors to probability vectors.

Subclasses override _estimate(counts_array) to return a numpy probability vector. The base class handles pandas-Series plumbing, normalization checks, and the describe() / __repr__ machinery.

typola.estimators.base.held_out_score(estimator: Callable[[ndarray | Series | Sequence[float]], ndarray | Series | Sequence[float]], counts_train: ndarray | Series | Sequence[float], counts_test: ndarray | Series | Sequence[float]) → dict[source]

Fit an estimator on counts_train and score it on counts_test.

Returns a dict of metrics:

  • log_likelihood — higher is better

  • perplexity — exp(-log_likelihood / N_test); lower is better

  • kl_to_empirical — KL(empirical_test || predicted); 0 is perfect

  • name — estimator’s reported name

typola.estimators.base.log_likelihood(p: ndarray | Series | Sequence[float], counts: ndarray | Series | Sequence[float], *, eps: float = 1e-12) → float[source]

Log-likelihood of multinomial counts under distribution p.

Returns sum(counts[i] * log(p[i])) ignoring the normalizing constant. Useful for comparing two estimators on the same held-out counts.
